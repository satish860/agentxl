Tester steps for Windows

If you are testing AgentXL and do not want to use npm or developer tools, follow these exact steps.

1
Download the latest release

Open the latest GitHub release and download the Windows asset.

2
Install or extract the Windows release

If you downloaded the MSI or EXE, install it. If you downloaded the ZIP, extract it to a normal folder such as Desktop\AgentXL.

3
Launch AgentXL onboarding

If you installed AgentXL, launch Launch AgentXL onboarding from the Start Menu. If you are using the ZIP, double-click Launch AgentXL Onboarding.cmd. This starts AgentXL, configures Office, and opens Excel.

4
If sign-in is needed, run login once

Use AgentXL Login from the Start Menu or AgentXL Login.cmd from the ZIP, complete the browser sign-in, then retry onboarding.

5
Manual fallback only if needed

If Office blocks the automatic setup on that machine, add the extracted manifest folder in Trusted Add-in Catalogs, restart Excel, then add AgentXL from SHARED FOLDER.

Download options

Choose the path that fits your tester or user.

Windows release download Best for testers. Look for AgentXL-Setup-*.msi, AgentXL-Setup-*.exe, or the portable AgentXL-Windows-Payload-*.zip. npm package Best for technical users. Install with npm install -g agentxl.
Do users still have to add the Excel add-in manually?

Usually no on Windows now. The new Windows flow tries to trust the certificate, register the add-in with Office, enable loopback, and open Excel directly. But keep a manual fallback for locked-down Office environments.

What AgentXL does

AgentXL is not a generic spreadsheet chatbot. It is built for work where:

  • supporting evidence lives in local documents
  • the final output belongs in Excel
  • the answer must be traceable back to source files

Best first test

Give your tester one workbook and one real folder of support files. Then ask them to try:

  • link a folder
  • ask a grounded question
  • inspect the answer
  • check where the answer came from

Example prompt

Extract the cash balance from the bank statement folder and map it to the cash workpaper.

That is the core experience AgentXL is built for.

Quick answers

Do I need a separate AgentXL account?

No. You use your own model provider credentials.

Do I need Node.js to run the Windows release zip?

No. The current Windows package is self-contained and includes its own Node runtime.

Do I still need to do anything manually in Excel?

Usually not on Windows now. Start with Launch AgentXL Onboarding.cmd. If Office blocks automatic setup on that machine, use the manual Trusted Add-in Catalog fallback.

Where do I download the latest build?

Use the latest release page: github.com/satish860/agentxl/releases/latest. Prefer the MSI when available, then the EXE, then the ZIP.
