AgentXL runs as two pieces: an Excel add-in (the UI in your taskpane) and a local server (processes your documents, calls the AI model). Your documents never leave your machine.

Publishing to the Office Store? See AppSource instructions ↓

1 Install the Excel add-in

Choose one of these methods to get AgentXL into your Excel ribbon:

Option A — Office Store (recommended)

  1. Open Excel
  2. Go to Insert → Get Add-ins (or Office Add-ins)
  3. Search for "AgentXL"
  4. Click Add
ℹ️ The Office Store listing is pending review. Use Option B or C in the meantime.

Option B — Sideload the hosted manifest

  1. Open Excel
  2. Go to Insert → Get Add-ins → My Add-ins
  3. Click Upload My Add-in
  4. Paste this URL or download the manifest:
    ⬇ Download manifest.xml
  5. Click Upload

Option C — Shared folder catalog (persistent)

  1. Excel → File → Options → Trust Center → Trust Center Settings
  2. Click Trusted Add-in Catalogs
  3. Add the manifest folder path from your downloaded release
  4. Check Show in Menu → OK → OK
  5. Restart Excel
  6. Insert → My Add-ins → SHARED FOLDER → AgentXL → Add

2 Download & run the local server

The server handles document processing, AI model calls, and folder scanning. Everything stays on your machine.

📦 Windows ZIP

Windows ZIP (no Node.js needed)

  1. Download the latest .zip from GitHub Releases
  2. Extract to a folder (e.g. Desktop\AgentXL)
  3. Double-click AgentXL.vbs
  4. First time? It opens sign-in — choose your AI provider
  5. Server starts silently. Excel opens with AgentXL loaded.

npm (cross-platform)

npm install -g agentxl
agentxl start
✅ The Windows ZIP bundles its own Node.js — no system install required.

3 Use it

  1. Open Excel and click AgentXL on the Home ribbon
  2. Link a local folder containing your source documents
  3. Ask a question — the agent searches the folder, reads the files, and returns grounded answers

Example prompts

"List all files in the folder and summarize what each contains"
"Extract the ending cash balance from the bank statements"
"Compare the trial balance to the lead sheet and flag mismatches"
"Pull lease terms from these agreements into a schedule"

Publishing to AppSource (for maintainers)

To make AgentXL installable via Excel → Insert → Get Add-ins, you need to submit the hosted manifest to Microsoft's Partner Center.

Prerequisites

  • A Microsoft Entra ID work/school account (not a personal @outlook.com or @hotmail.com)
  • Authority to sign legal agreements on behalf of your organization
  • The taskpane must be hosted at a public HTTPS URL (GitHub Pages is set up for this)
⚠️ Personal Microsoft accounts won't work. You need a work/school account (Microsoft Entra ID). If you don't have one, you can create a free tenant at Microsoft 365 Developer Program.

Step 1 — Enroll in Partner Center

Go directly to the Office enrollment page:

https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/office

Do not use partner.microsoft.com/dashboard — that generic URL often fails or redirects incorrectly. Use the direct enrollment link above.

  1. Sign in with your work/school account
  2. Create a new work account or link an existing one
  3. Fill in your publisher profile (company name, contact info)
  4. Accept the Microsoft Office Agreement

No fee. Office add-in publishing on AppSource is free.

Step 2 — Enroll in the Microsoft 365 and Copilot program

If you already have a Partner Center account from another program:

  1. Go to Settings → Account Settings → Programs
  2. Find the Microsoft 365 and Copilot tile → click Get Started
  3. Create or select a publisher account
  4. Accept terms

Step 3 — Enable GitHub Pages

Before submitting, the taskpane must be live:

  1. Go to your GitHub repo → Settings → Pages
  2. Source: GitHub Actions
  3. Push to main — the workflow deploys automatically
  4. Verify https://satish860.github.io/agentxl/taskpane/ loads

Step 4 — Submit the add-in

  1. In Partner Center, go to Marketplace offers
  2. Create a New offer → Office Add-in
  3. Upload manifest-hosted.xml (from this repo's manifest/ folder)
  4. Fill in metadata:
    App nameAgentXL
    Short descriptionTurn source documents into traceable Excel workpapers
    CategoryProductivity
    Support URLhttps://github.com/satish860/agentxl/issues
  5. Upload 2-3 screenshots of the taskpane in Excel
  6. Provide a privacy policy URL
  7. Submit for review

Step 5 — Wait for review

Microsoft reviews the submission (typically 4-6 weeks for first submission, faster for updates). They check:

  • Manifest validates correctly
  • Taskpane URL loads and is functional
  • Basic UX quality and accessibility
  • No malware or privacy violations
  • Graceful handling when the server is offline (your ConnectionError screen handles this)
✅ Once approved, AgentXL appears in Excel → Insert → Get Add-ins for all users worldwide. No sideloading, no Trust Center, no manifest files.

Troubleshooting

Taskpane shows "Start AgentXL first"

The server isn't running. Double-click AgentXL.vbs or run agentxl start.

Certificate warning in browser

npx office-addin-dev-certs install

Then restart Excel.

Add-in doesn't appear after sideloading

Close and reopen Excel. Check Insert → My Add-ins.

Port 3001 already in use

agentxl start --port 3002
